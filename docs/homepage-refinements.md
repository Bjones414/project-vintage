TopNav now conditionally renders the URL paste field: on `/` the field is hidden (the hero form is the entry point), and on all other pages (`/analyze/[id]`, future routes) the field remains visible as the primary way to start a new analysis from a report page. The avatar cluster retains its `ml-auto` on all viewports when the field is absent so logo and avatar stay at opposite ends on home; on non-home pages the existing `sm:ml-0` override keeps the three-item desktop layout intact. The homepage subhead received `text-balance` (CSS `text-wrap: balance`, available in Tailwind 3.4) and its max-width was widened from 520px to 580px, giving the browser more room to find natural phrase boundaries — "generation-level watch-outs" no longer splits mid-phrase. TopNav tests were updated: the submit/error scenarios now run against a non-home pathname so the URL field is present, the stale `/analyze` return-null test is removed, and a new assertion verifies the field is absent on `/`.
