### CHECK — both sections, discrepancies to resolve

1. **Gmünd production figure conflict:** `generation-content.ts` says "approximately 60 total"; `canonical-porsche-production.md` (citing elferspot.com / Porsche AG) says 52. Resolve against 356 Registry chassis ledger — 52 is the more conservative, better-cited figure.

2. **356 A Carrera engine span:** `356a:carrera` in factory-specs.ts specifies "1.5L Type 547/1 GS Carrera four-cam, 100 hp." The 356A Carrera actually ran two engine generations: 1.5L Type 547/1 (1955–57) and 1.6L Type 692/0 (from 1958). The catalog entry conflates these into one spec. Consider splitting into `356a:carrera_1500` and `356a:carrera_1600`, or annotating the engine transition at mid-A production.

3. **356 B 1600 Super scope:** `356b:1600_s` (75 PS) in factory-specs.ts is listed without T5/T6 date bounds. Canonical distinguishes T5 (1959–1961) as the Super era and T6 (1961–1963) as the S90 era (the Super 90 largely superseded the plain Super in T6). The catalog entry should be scoped to T5 years only to avoid incorrect comp matching for T6-era cars.

4. **356 B Carrera 2 production count:** Catalog (generation-content.ts) and canonical-porsche-production.md both flag ~310 [VERIFY]. Sources range 280–436. The 356 Registry is the primary authority; update when confirmed. Do not propagate this figure into display copy without the caveat.

5. **HP standard labeling:** All HP values in factory-specs.ts for pre-1972 models are DIN PS, but the field is labeled "hp". The difference is <2% (1 PS = 0.9863 hp SAE) and immaterial for comp-engine matching, but any display output that renders "130 hp" for the Carrera 2 should clarify the standard to avoid misrepresentation in user-facing copy.

6. **956 B 1600 S T5/T6 body notation:** The factory-spec entry `356b:1600_s` covers "Coupé, Cab" but does not flag that the T5 Coupé and T6 Coupé have distinct visual proportion. The T5 (lower beltline) commands a small premium over the T6 in the market. The comp engine should not treat them as identical if year data is available on listings.

7. **356 A Speedster production split:** The 2,910 Speedsters attributed in the catalog (`356a:speedster`, production drawn from generation-content.ts) are the total 356A Speedsters across all engine specs (1600 Normal and 1600 Super were both available). A single catalog entry is reasonable, but value-driver logic should account for the ~10–20% premium an original 1600S-engined Speedster commands.

8. **912 project scope conflict:** `PROJECT_STATE.md` line 360 lists "912 — explicitly excluded." However, `docs/overnight/trim-list-audit.md` (2026-05-18) recommends adding the 912 to models.ts. These conflict. Scope decision must be made and documented in PROJECT_STATE.md before any catalog work on the 912 begins.

9. **914 project scope conflict:** `PROJECT_STATE.md` line 360 lists "914 — explicitly excluded." Section B documents the full canonical 914 lineup for reference. No catalog action should be taken without a scope decision. Adding 914-6 GT (RACE-ORIGIN) would also require a policy decision on race cars.

10. **models.ts authority gap — 356 specifically:** `factory-specs.ts` has 18 entries and `generation-content.ts` has 4 generation sections for the 356, but `models.ts` has zero GENERATION_DEFS entries. This means any 356 listing in the DB with a `generation_id` (e.g., `'356-pre-a'`) will have no valid trim combinations available from the catalog's runtime authority, and `isValidCombination()` will return false for all year/trim pairs. The DB generations seed reportedly has 356 rows; the models.ts gap must be closed for the comp engine to function for these listings.

11. **trim-category/index.ts unhandled:** No `derive356()`, `derive912()`, or `derive914()` function exists. All these generation IDs fall through to `default: return null`. This is acceptable behavior for the comp engine's D6 rule (which skips trim-category filter when null), but it means no trim isolation will occur for these listings until handlers are added.

12. **912E US-only flag:** All 2,092 912E units were destined for the US market. If added to the catalog, this model should be flagged as US-only to prevent incorrect comparable matching with European auction data.

13. **914 total production arithmetic:** The sub-variant estimates (1.7: ~74,000; 1.8: ~25,000; 2.0: ~17,000; 914-6: 3,349) sum to ~119,349, broadly consistent with the commonly cited ~119,000 total. The individual variant estimates are rounded; do not treat them as confirmed Porsche AG figures.

14. **914-6 US allocation:** ~1,000 US units for the 914-6 is a [VERIFY] estimate. Total production is well-documented at 3,349; the market-by-market split is not publicly granular from Porsche AG. PCA membership records or registry data would be the authoritative source.

15. **912 Targa production split:** Total 912 (~30,300) is well-cited (PCA); the Coupé/Targa split (~27,700 / ~2,600) is estimated from the Targa body's 1967 introduction leaving only three model years of Targa production. This split should be treated as approximate until a registry-level source confirms it.

16. **356 Pre-A 1500 GS Carrera count conflict:** generation-content.ts says "~100 cars (combined pre-A and early 356 A production)"; canonical-porsche-production.md says "~100 [VERIFY]". Both are consistent but uncorroborated. The count of pre-A-era Carrera cars (1955 only, before 356A launched mid-year) may be as few as 15–30 cars. [VERIFY against 356 Registry].

17. **356 B Carrera GTL Abarth race-origin classification:** The 21-unit Abarth-bodied Carrera B was sold to customers and road-registered in some markets. Porsche's own literature lists it as a production model, not a works race car. However, it was purpose-built for GT racing. If a RACE-ORIGIN flag tier exists in the catalog, this should be classified as RACE-ORIGIN-ADJACENT (customer competition car, road-registerable).

18. **generation-content.ts partial read:** The file at 451.9 KB exceeds the 256 KB single-read limit used in this session. All mechanical-issue flags in Section A for 356 trims were extracted from partial reads. A full read via grep-based extraction of the 912 and 914 sections confirmed those sections do not exist. Recommend a targeted read of the complete 356-pre-a/356a/356b/356c sections before any production data in generation-content.ts is treated as authoritative.
