The four-step loading state step timers in `components/analyze/AnalyzeLoadingState.tsx` were tightened from `[2000, 5000, 9000, 12000]` ms (total ~12 s, ~3 s per step) to `[1125, 2250, 3375, 4500]` ms (total 4.5 s, ~1.125 s per step). The wait-for-both behavior is unchanged: if the API responds in under 4.5 s the redirect fires the moment step 4 completes; if the API is slower, the redirect waits for it. Two timer-dependent assertions in `tests/components/analyze/AnalyzeLoadingState.test.tsx` were updated to match the new boundary values (`1126` ms for the step 1 → step 2 transition, `4501` ms for the step 4 completion).
